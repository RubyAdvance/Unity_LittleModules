# Unity_LittleModules
Unity常用系统归纳

unity版本：2021.3.32f1c1

工程项目的描述和提示可以结合场景中的Tips以及游戏系统汇总.Xmind文件一起理解。

1.角色受击闪白闪红冰冻效果，包含序列帧角色和spine角色，总共总结了3种实现思路。分别取自G89、G119、AI实现
场景：
角色受击闪白闪红冰冻效果-方案1
角色受击闪白闪红冰冻效果-方案2
角色受击闪白闪红冰冻效果-方案3
主要核心写一个Shader曝露出修改的属性值，然后创建材质球复制给角色，通过代码控制材质球的Shader属性值控制角色的闪白闪红效果。

2.引导系统：主要是实现遮罩效果，即强引导效果，总共有两种实现方案。
方案1（）：取自G89,主要是利用材质镂空挖空，就是将图片的一些像素alpha值设置为0实现镂空效果，然后收到点击事件的实际从点击的位置将点击事件
透传到UI下方实际的目标UI，这样就触发了实际了UI的点击事件。但是这种在多个镂空区域的时候不是很好用，多个镂空区域倾向于方案2，且这个方案相对复杂。
方法2：取自G102的实现思路，多个Shader材质交互实现镂空区域，镂空区域作为一个按钮接收点击事件，然后再调用时机的UI点击事件。多个镂空区域实现起来很友好。

3.无限循环列表（无限循环列表-AI实现）:实现了竖屏和竖直方向上的无限循环，是来自AI的实现，G89实现的太过复杂，G119实现的没有解耦（但是需要的话可以参考改动较小）
另外实现ScrollRect慢慢滚动到固定的位置也制作了一个场景：ScrollView滚动测试。

4.签到系统（签到系统）：制作了一个签到系统处理模板，实现了签到时间的处理和签到补签等等功能，以及时间的转换显示。可以复用，相对方便快捷。

5.摇杆移动2D/3D游戏对象（摇杆移动2D3D游戏对象）：制作了一个场景，场景中包含2D和3D游戏对象，挂载脚本兼容实现了2D和3D游戏物体的摇杆移动。
